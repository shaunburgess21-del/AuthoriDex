------------------------------------------------
Goal
------------------------------------------------

Add a **prediction feature (test mode only)** into the existing FameDex app, with good UX on both **mobile and desktop**, while preserving the current design and page structure.

Context (short):
- FameDex already has:
  - A Home page with leaderboards and a “Cast Your Vote” sentiment slider.
  - A Person Profile page with Trend Score, charts, platform insights, etc.
  - Supabase for backend/auth.
- We now want to add a **“Predict”** experience:
  - Users can “predict” if a person’s Trend Score will go up or down for the week.
  - Initially this is **test mode only** with mock data / virtual credits (no real money, no smart contracts yet).
- The UI must work well on **mobile first**, but also on desktop.

You must work WITH the existing components and layout patterns. Extend them, don’t replace them.

B. NEW PREDICT PAGE – /predict
------------------------------------------------

Create a new route/page at **`/predict`** that lists prediction markets using MOCK DATA.

Requirements:
- Title: **“Prediction Markets”**
- Subtext: “Back your insight on who’s really moving the world. Currently running in test mode with virtual credits.”

- Below that, show a vertical list of **market cards**. Use a hard-coded array of mock market objects for now.

Each market card:
- Person avatar + name (e.g. Elon Musk)
- Small label: “Weekly Trend Score · Ends Sun 23:59 UTC”
- Optional mock current Trend Score or 7d change
- Two pill-style badges/buttons on the right with mock odds, e.g.:
  - “Up 1.7×”
  - “Down 2.3×”
- CTA / chevron: “View Market”

Click behaviour:
- Clicking a card should open a **Market Detail view**. For v1 you can either:
  - Navigate to `/predict/[marketId]`, OR
  - Navigate to that person’s profile page and auto-select its **Predict** tab (see section C).
- Choose whichever best fits the existing routing structure and keep the code small and additive.

Styling:
- Follow existing card styles and spacing.
- On mobile: full-width cards in a simple column.
- On desktop: centered content with the same max-width/container as the existing pages.
