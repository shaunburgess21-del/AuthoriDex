Prompt for Vote and Predict Toggles on Home Page

I now want to move onto how we feature the Vote and Predict sections on the main home page. Currently - on the home page - we have 3 toggles (Overview | Vote | Predict). The purpose of the toggles is to mitigate scroll fatigue while we can still represent the 3 main features of FameDex - namely: Leaderboard, Vote and Predict. We have got the Overview (leaderboard) toggle display looking good; however, I want to optimise how the Vote and Predict displays appear and function under their respective toggles. See attached screenshots - currently the Vote and Predict toggles show a snippet of what you can find on the Vote or Predict pages with a button at the bottom which then takes a user to either the vote or predict page to view more / all. For reference - the button at the bottom of page with Predict toggle selected is "Explore All Prediction Markets" and the button at the bottom of page with Vote toggle selected is "Go to Governance Hub". 

The idea to optimise - which applies to both the Predict and Vote toggles - is the following: Have a heading for each category that you can find on the vote or Predict page with one card below each heading - and have a tinder (online dating app) style way of interacting with each card. You won't swipe left or right - but rather interact with the card as you would on the main vote or predict page by selecting a vote/predict button on the card and as the user selects the vote or predict button - then that card disappears and a new card appears for them to interact with and so on. 

For context - Replit has actually already built this functionality into the website for the "Curate Profile" category on the "Vote" page itself. So, we want to ask Replit to reference this same principle when applying this new idea to the Home page Vote and Predict toggles.
