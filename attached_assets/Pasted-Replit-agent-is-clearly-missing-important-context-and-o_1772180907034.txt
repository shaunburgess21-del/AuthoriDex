Here’s my take on the actual red flags from the Mediastack review, and what I’d tell Replit to fix/check next.

What is actually a red flag
1) Persistent high zero-article coverage from Mediastack

This is the real issue.

If, on refresh runs, you’re seeing:

nearZeroPct = 71–82%

lots of people with 0 articles

recurring offenders in the top 25

then Mediastack is operationally up but data-quality weak for your use case.

That means:

the API is “healthy” technically,

but your news signal is underpowered for a meaningful chunk of the leaderboard.

This matters more than the single timeout.

2) Repeat zero coverage for important names

Also real.

If names like:

MrBeast

Lisa / Blackpink

Ed Sheeran

Theo Von

Nayib Bukele

keep getting zeroes, that means Mediastack has blind spots by:

entertainment/music

international figures

certain creator/political profiles

That is exactly the kind of bias that can distort a “who’s trending” product.

3) Duplicate runs in the same hour bucket

This is worth investigating.

If you truly had:

one fresh run

one cache-only run

both completing for the same hour bucket

then one of these is happening:

the lock is not preventing overlap correctly

a retry/backfill path is writing into the same hour bucket

the “fresh vs cache” decision is being recomputed in a second invocation for the same bucket

the ingestion scheduler is firing twice

Even if snapshots are idempotent, this can still:

skew health metrics

confuse provider stats

produce inconsistent “last run” diagnostics

This is a real ops bug to inspect.

What is not necessarily a red flag
4) newsGovernor = 75% on cache-only runs

I would not call this a bug by itself.

That sounds like your freshness governor doing exactly what it was designed to do:

fresh fetch = full weight

cache reuse = dampened weight

That is a feature, not a failure.

The only question is whether 75% is the right penalty. But “it happens every cache hour” is expected if your cadence is intentionally:

refresh every 2h

cache on the in-between hour

So I would tell Replit:

This is not a red flag unless the penalty is making news too weak in practice.

5) One timeout in 48h

One isolated connection timeout is acceptable if fallback worked and the pipeline stayed healthy.

That’s not what I’d focus on.
