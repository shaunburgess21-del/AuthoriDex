Replit Prompt: Opinion Polls UI/UX overhaul to match Sentiment Polls standard

We need a drastic improvement by reusing proven layout/components from Sentiment Polls and adapting only what’s necessary for multi-option voting.

Move the category label (e.g. “Tech”) to the top-right corner of the card (not top-left).

Apply the correct category color styling exactly like all other cards across Vote sections (Tech/Business/Politics/etc).

A2) Primary avatar beside the title

On the card header, display the poll’s main avatar image to the left of the title (consistent with other card patterns).

If no main avatar image exists, use the same placeholder rules you use elsewhere (not a new one).

A3) Avatar shape consistency

All avatars in Opinion Polls (card + detail page) must be square with rounded corners (same radius as elsewhere).

Remove circular avatars entirely.

A4) Show 4 options by default (not 3)

On the Opinion Poll card, display 4 options by default.

If there are more than 4, show “+X more options” text below (dynamic count).

A5) Inline voting must work from the card (no forced navigation)

Current behavior is wrong: clicking an option opens the detail page. We need:

Clicking an option row’s Vote button should cast the vote immediately from the card.

Do NOT navigate away when voting from the card.

After voting:

show results on the card for the visible options (the 4 shown)

show % (and/or bars) for those visible options, consistent with your Results styling

Navigation rules:

Clicking the title opens the detail page

Clicking “+X more options” opens the detail page

Clicking whitespace on the card (optional) can open detail page, but vote buttons must not.

The Opinion Poll detail page currently feels like a basic list. We want it to mirror the structure and polish of the Sentiment Poll detail page, adapted for multi-option voting.

B1) Header section (copy Sentiment Polls structure)

Match Sentiment Poll detail header layout:

category chips/tags styling and placement

title + description

B3) Results widget (beautiful + consistent)

Add a dedicated Results component that looks and feels like Sentiment Polls Results:

bars / percentages

B4) Context + Discussion (keep consistent)

Keep the Context block styling consistent with Sentiment Polls.

Keep the Discussion component exactly the same component used on Sentiment Polls detail pages.

Do not invent new UI patterns. Reuse the existing Sentiment Polls components wherever possible:

Card header layout patterns

badges / category chips

avatar styling

results bar styling

typography, spacing, shadows, borders

Opinion Polls should look like it shipped with the rest of AuthoriDex from day one.

Main poll avatar displays left of the title on card.

All avatars are square with rounded corners (no circles).

Card shows 4 options by default + “+X more options”.

Detail page layout matches Sentiment Poll detail page structure:

Vote widget near top

Results widget with bars/percentages and total votes
