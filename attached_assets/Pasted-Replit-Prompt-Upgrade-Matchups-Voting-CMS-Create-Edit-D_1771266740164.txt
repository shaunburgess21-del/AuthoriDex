Replit Prompt: Upgrade “Matchups” (Voting CMS) Create/Edit + Dedicated URL (Non-Breaking)
Goal

Improve the Voting CMS → Matchups create/edit modal and the frontend matchup cards with:

celebrity autocomplete + auto-avatar linking

optional image upload for non-leaderboard options

visibility statuses + featured toggle

slug + dedicated URL page opened only by clicking the card heading (not vote area)

This must be implemented without breaking existing matchups already listed in the admin panel.

A) Confirm existing data model first (important)

Before changing anything, inspect the existing Matchups schema/model/API. There is already a Matchup Queue with Active items, so this likely already has fields like:

id, title, category, optionA, optionB, isActive (or status), createdAt, etc.

Do not rename existing fields if they are already used. Only add new fields as nullable so existing data continues to work.

B) Data model extensions (additive / safe)
Add these new fields to the Matchups table (all optional/null by default)

visibility (enum/string) with allowed values:

draft (Admin only)

live (Active)

inactive (Visible but dimmed)

hidden

archived

featured (boolean, default false)

slug (string, unique, nullable initially — enforce uniqueness on non-null)

optionACelebrityId (nullable FK / string id)

optionBCelebrityId (nullable FK / string id)

optionAImageUrl (nullable string) ← for custom uploads

optionBImageUrl (nullable string) ← for custom uploads

(keep existing preVotePrompt field if it already exists)

Non-breaking rule

If existing schema already has something similar (e.g. isActive), map it:

isActive=true → visibility=live

isActive=false → visibility=inactive (or draft depending on current usage)

Do a simple backfill migration so existing records get a valid visibility.

C) Resolved avatar logic (API response)

In the Matchup API response, return resolved avatar URLs so the frontend doesn’t need to guess:

optionAAvatarUrl = celebrity.imageUrl if optionACelebrityId is set and exists else optionAImageUrl else fallback

optionBAvatarUrl = celebrity.imageUrl if optionBCelebrityId is set and exists else optionBImageUrl else fallback

Also return:

visibility, featured, slug

This should work for existing matchups even if none of the new fields are filled.

Draft (Admin only)

When typing in Option A or Option B:

query leaderboard/celebrities endpoint (like Linked Celebrity field)

set optionAName / optionBName to the celeb name

disable the custom image upload OR keep enabled but show “Using celebrity image” and ignore custom image unless celebrity cleared

4.2 Custom avatar upload for non-leaderboard options

Add an Upload Image button for each side:

“Upload image for Option A”

“Upload image for Option B”

Flow:

Uploads to Supabase Storage

Auto-fills optionAImageUrl / optionBImageUrl

Show preview thumbnail in modal

If a celebrity is selected, use celebrity avatar by default

If celebrity is not selected, use uploaded image if provided

On the frontend Matchup cards:

Clicking the main heading/title text opens the dedicated matchup URL: /vote/matchups/:slug

Clicking on the vote area (images/buttons) must only vote, and must NOT navigate.

This matches the Real-World Markets / Trending Poll intent: fast voting from cards, deeper info from heading.
