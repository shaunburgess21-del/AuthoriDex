You’re seeing a real UX inconsistency, and Replit’s diagnosis is basically correct: Hot Movers and the Leaderboard are currently “classifying” movement using different rule systems, so the same person can end up with different labels/icons in each place.

What’s actually happening (why Jake Paul can be “Spiking” in Hot Movers but “Breakout” on the leaderboard)

Hot Movers: typically uses simple fixed thresholds on 24h % change (e.g., ≥ +25% = “Spiking”, ≤ −15% = “Cooling”). This is a single-metric classifier.

Leaderboard icons: uses percentile-based + multi-metric logic (often % change AND rank change), so someone can qualify as:

Breakout (🚀) = big % change and big rank jump (top percentile on both)

Spiking (⚡) = big % change only

Cooling (❄️) = big negative move (and maybe also rank drop), etc.

So Jake Paul can clear Hot Movers’ “Spiking” threshold and also meet the leaderboard’s stronger “Breakout” criteria—hence different labels.

And someone can be “Cooling” in Hot Movers (because −15% crosses the fixed threshold) but not get a snowflake on the leaderboard if they didn’t also meet the percentile/combined rule.

Should you “unify the labeling” like Replit suggests?

Yes—but not in the extreme “make everything identical everywhere” way. You want consistency without losing information.

There are two good options:

Option A (best): Make Hot Movers show the same classification as the leaderboard

Keep Hot Movers as “Top % gainers · 24h” (ranking stays by %), but the badge/icon shown next to each person should come from the same getExceptionalIndicator() logic as the leaderboard.

If someone is 🚀 Breakout on the leaderboard, they should show 🚀 in Hot Movers too.

If someone is ⚡ Spiking, show ⚡.

If they don’t qualify for any exceptional badge, show nothing (or a neutral label like “Mover”).

This preserves:

Consistency (same person → same badge)

Meaning (Breakout remains “stronger than spiking”)

Important: Hot Movers can still be sorted by % change; the badge is just a classification layer.

Option B (also valid): Keep Hot Movers as “simple labels”, but rename them so users don’t expect the same system

If Hot Movers uses fixed thresholds, then don’t call it “Spiking/Cooling” with the same icons you use elsewhere. Call it:

“Big % gainers” / “Big % losers”

or “Up big” / “Down big”
and use different visuals.

This avoids collision, but you lose the nice unified vocabulary you’ve built.

My recommendation to share with Replit

Tell them to implement Option A:

Hot Movers should reuse the exact same indicator function and thresholds as the leaderboard (percentile-based, multi-metric if that’s how you define Breakout/Cooling).

Hot Movers sorting remains % change (because it’s “Top % gainers · 24h”).

If you still want Hot Movers to display “Spiking / Cooling” text, derive that text from the same classification outcome (e.g., 🚀 → “Breakout”, ⚡ → “Spiking”, ❄️ → “Cooling”, 📈 → “Rising”).

That way:

Jake Paul shows 🚀 in both places (if he truly qualifies as Breakout).

“Cooling” in Hot Movers only appears if ❄️ would also appear on the leaderboard—no more “why doesn’t the board show it?”

“Badge meaning: Breakout = big % move + big rank jump”
