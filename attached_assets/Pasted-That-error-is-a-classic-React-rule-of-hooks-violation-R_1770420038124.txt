That error is a classic React rule-of-hooks violation:

“Rendered more hooks than during the previous render.”
It happens when a useState / useEffect / useCallback / useMemo (any hook) is being called conditionally (inside an if, switch, return early, inside a tab render, etc.). On one render React “sees” N hooks, on the next it “sees” N+1 hooks → crash.

Your screenshot points at AdminDashboard.tsx:1057 where searchCelebrities = useCallback(...) lives. That means Replit likely declared this hook inside a conditional block (e.g., only when the “Trending Polls” tab is active, or only when modal open, or only after auth loads).
