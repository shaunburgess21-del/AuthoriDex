Below is the clean way forward.

1) Search Interest: yes, you can show “popular searches” — but only if you store them
What you can get (cheaply, from what you already use)

From Serper, you can usually get:

relatedSearches (best for “Top searches”)

peopleAlsoAsk (great for “People ask”)

So in principle: yes, you can show the top searches per person with no extra provider.

Why it’s empty right now

One of these is happening:

You didn’t persist relatedSearches / peopleAlsoAsk from Serper into your cached result / snapshot diagnostics, or

You did, but the cached Serper entries were created before the new fields existed, so you won’t see them until you force a refresh / cache bust / wait for fresh ingests, or

You’re fetching Serper but only using it to compute a score, then discarding the raw query lists before the momentum endpoint reads them.

Fix (best practice)

Make “Search Interest” always show two layers:

Layer A (always available):

“Search activity score: 69/100”

“Trend: flat / +x% / -x%” (when you have a meaningful delta)

Layer B (only when available):

Top searches: 3–5 relatedSearches

People ask: 2–3 questions

If Layer B is missing, don’t leave it blank — show:

“Top searches: not available yet (collecting data)”
This keeps trust high and avoids the “empty card” feel.

Quick win if Serper query lists remain sparse

If relatedSearches/PAA are sometimes empty for certain people:

Add a fallback that generates “topics” from your existing news headlines (cheap keyword extraction). Even 3 bullets like:

“jaw surgery”

“Anthony Joshua”

“Winter Olympics / Jutta Leerdam”
…makes the card feel alive without new APIs.

2) Score Drivers: what makes it interesting to users?

Score Drivers is not interesting as “40% news, 30% search, 30% wiki” (those are just your model weights).

It’s interesting when it answers:

“Why did this person move today?”

So the user-facing version should be:

Drivers of the 24h move (attribution), not static weights

Example:

“62% Search surge”

“28% News volume”

“10% Wikipedia views”

That is screenshot-worthy because it explains the move.

Why yours still shows 0/0/100

Your current driver computation is likely based on:

snapshot deltas (searchDelta, newsDelta, wikiDelta)

and/or EMA-smoothed signals
…and in this case, Search + News deltas are effectively near-zero (or filtered away), leaving only Wiki.

That can happen even when 24h change is big if:

news/search are contributing mostly to mass (level) rather than velocity (change), or

your delta pipeline is still stabilizing (searchDelta only recently got fixed), or

you’re attributing from the wrong layer (raw deltas instead of score-component deltas).

Fix (the robust, simple version)

For Score Drivers (24h change), compute attribution from the same things that create the 24h change:

Option A (best): attribute the score change itself

Compute:

Δscore_from_search

Δscore_from_news

Δscore_from_wiki
between “now” and “~24h ago” using your real scoring transforms

Then:

pct_i = abs(Δi) / sum(abs(Δall))

This is the cleanest and avoids weird 0/0/100 artifacts.

Option B (ok for launch): attribute from normalized signal deltas

Use normalized deltas, but:

include denominator floors

clamp

and hide if movement is trivial

Either way, when only one signal is present, you should say that:

“Drivers based on 1/3 sources (Search & News quiet)”

That turns a confusing chart into an honest insight.

Gating rule

Replit’s idea to gate primarily on abs(change24h) is correct for trust.

But if you still see 0/0/100 on people with big 24h change:

that means your attribution inputs aren’t aligned with the change.
So don’t just tweak gating; fix attribution (Option A above).

3) Will Momentum Signals be “fresh” for the bottom half of the leaderboard?

Freshness and activity are different.

If your ingestion runs hourly for all 100 people, then the data timestamp is fresh for everyone.

But many lower-ranked people will have:

0–1 news articles

flat search score

low wiki views
So their cards will look “quiet,” not “stale.”

How to make “quiet” still feel good

Add copy + UI states:

If News = 0:

“No major headlines in the last 24h”

If Search is flat:

“Search interest steady”

If Wiki is low:

“Low curiosity signal today”

This makes the bottom half feel intentionally calm rather than broken.
