7) Weekly Reset + Seeding System (Critical)

We need to avoid the “empty restaurant” problem at launch.

Weekly reset rules:

All native prediction sections reset weekly at Sunday 23:59

New weekly cycle begins Monday 00:00

Seed activity should accumulate mostly Mon–Tue, in small hourly batches, to look organic.

What to seed:

For each market type:

Participants count

Distribution across outcomes (Up/Down; A/B; Top Gainer picks; Jackpot guesses)

Total pool (credits)

How to seed:

Create a “Seed Engine” that runs automatically:

Runs hourly from Monday 06:00 to Tuesday 22:00 (configurable)

Adds small batches each hour until a weekly target is reached per market/card

Targets should vary per card so it doesn’t look uniform

Admin seed configuration fields:

For each card/market, add seed config fields (with sensible defaults):

Weekly target participants (min/max range)

Weekly target total pool (credits) (min/max range)

Distribution bias (optional):

Head-to-Head: bias % A vs B

Top Gainer: weighted distribution across selected celebs

Jackpot: generate guesses around plausible Trend Score range (based on last known Trend Score + volatility band)

Important:

Seed activity must look like “real users”:

Use pseudo-user identities (system/bot users) OR a “seed” flag in bets table

Ensure UI counts include seed activity

Prevent seeded votes/bets from being editable by real users

Weekly close + settlement:

At Sunday 23:59:

freeze markets

determine outcomes based on Trend Score changes

settle automatically (preferred), but keep manual settle button in admin
